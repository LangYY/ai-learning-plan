# AI Learning Plan

一个 Claude Code / Claude 技能（skill），充当你的 AI 学习路径教练。你说出最近想学的 AI 方向，它把目标收窄成具体可达的路线，从一份可信信源白名单里逐条推送学习资料，在你学完后主动问感想、帮你记笔记，并把学习目标、路径、历史与笔记沉淀进本地 Markdown 文件。可同时维护多个学习目标，每个目标一个文件。

本技能只服务 AI / 机器学习相关方向。

## 它能做什么

- 收窄目标：把"想学大模型"这类宽泛方向，通过一两轮提问收敛成有边界的具体目标，并按你的基础校准难度。
- 规划路径：给出 3 到 6 个循序渐进的阶段，从直觉到机制、到动手、再到前沿与应用。
- 逐条供料：默认一次一条（数量可调），每条注明信源、可点链接、为什么排在这个位置、重点看什么。
- 信源纪律：只从优质信源白名单取材，主动避开内容农场、营销号、无出处的二手解读。
- 复盘记笔记：你学完后它先问感想，再把你的原话笔记原样沉淀进文件。
- 推进灵活：支持自评跳阶，也可由教练动态评估某阶段是否已熟练，从而压缩或跳过。
- 多目标管理：随时切换学习目标，各自独立成文件，互不干扰。
- 笔记同步（可选）：可镜像到你惯用的笔记软件（如 Notion）；检测到对应 MCP 或可写路径时写入，检测不到时如实说明并退回本地。

## 工作流程

1. 认路：进入时先看已有计划，判断这次是开新目标、续旧目标、报告学完，还是查看进度。
2. 确定目标并校准当前水平。
3. 设计学习路径，并与你确认或调整。
4. 建立计划文件（可选配置笔记同步目标）。
5. 供料：以联网搜索为主、模型知识兜底，从白名单逐条给资料。
6. 复盘：你学完后主动问感想，记录笔记，更新进度。
7. 管理：切换与维护多个学习目标。

## 信源策略

核心原则：有出处、第一手、作者可信；按"人/号"白名单，而非按平台。完整名单见 [references/sources.md](references/sources.md)，可自行增删。

- 白名单：GitHub 仓库、YouTube 频道、X 优质博主、arXiv 与各家官方博客，以及中文优质信源（B站头部 up 主、知乎可信作者、Datawhale 等）。
- 谨慎（仅作线索）：少数可信的新闻号与社区，用于追动态、找原始论文，不作为终点学习资料。
- 黑名单：内容农场、SEO 水文、营销号、标题党、无出处的二手解读、小红书等信噪比过低的来源。

## 文件结构

```
ai-learning-plan/
├── SKILL.md                 主流程与触发逻辑
├── references/
│   └── sources.md           信源白名单 / 黑名单（可编辑）
└── assets/
    └── plan-template.md     学习计划文件模板
```

学习计划的状态文件默认写在 `~/ai-learning/` 下，每个学习目标一个 Markdown 文件，由技能在运行时创建，不随本仓库分发。

## 安装

将本仓库克隆到个人 skills 目录：

```bash
git clone https://github.com/LangYY/ai-learning-plan.git ~/.claude/skills/ai-learning-plan
```

随后在 Claude Code 中说出最近想学的 AI 方向（例如"我最近想学 RAG"），即可触发本技能。
